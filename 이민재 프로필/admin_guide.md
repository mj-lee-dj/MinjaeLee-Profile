# 관리자 페이지 사용 가이드 (Admin Guide)

이 문서는 포트폴리오 웹사이트의 관리자 페이지(`admin.html`)를 사용하여 내용을 수정하고, 변경사항을 실제 사이트에 반영하는 방법을 설명합니다.

## 1. 개요
이 웹사이트는 **정적 사이트(Static Site)**로 구성되어 있어, 관리자 페이지에서 내용을 수정하더라도 '저장' 버튼을 누르지 않으면 **새로고침 시 내용이 사라집니다.**
변경사항을 실제 사이트에 영구적으로 반영하려면 **GitHub 저장소에 변경내용을 업로드(Commit & Push)**해야 합니다.
관리자 페이지의 **"☁️ GitHub에 저장"** 기능을 사용하면 이 과정을 자동으로 처리할 수 있습니다.

## 2. 준비물: GitHub Personal Access Token (PAT)
GitHub에 자동으로 저장하기 위해서는 **권한(Token)**이 필요합니다. 최초 1회만 입력하면 브라우저에 저장됩니다.

1. [GitHub 토큰 설정 페이지](https://github.com/settings/tokens)로 이동합니다.
2. **Generate new token (classic)**을 클릭합니다.
3. **Note**에 토큰 용도(예: `Portfolio Admin`)를 입력합니다.
4. **Select scopes**에서 **`repo`** 박스를 체크합니다. (데이터를 저장하기 위해 필수)
5. 맨 아래 **Generate token** 버튼을 클릭합니다.
6. 생성된 토큰(ghp_...로 시작)을 **복사**합니다. (화면을 벗어나면 다시 볼 수 없으니 주의하세요!)

## 3. 사용 방법

1. **관리자 페이지 접속**: `admin.html`을 엽니다.
2. **내용 수정**: 대시보드나 각 탭에서 내용을 추가/수정/삭제합니다.
3. **저장하기**:
   - 우측 상단의 **"☁️ GitHub에 저장"** 버튼을 클릭합니다.
   - (최초 1회) 복사해둔 **GitHub 토큰**을 입력창에 붙여넣고 확인을 누릅니다.
4. **완료 확인**:
   - 화면 하단에 **"✅ 저장 성공!"** 메시지가 뜨면 완료된 것입니다.
   - 이미지가 포함된 경우 업로드 시간이 조금 걸릴 수 있습니다.

## 4. 변경사항 확인
저장이 완료되면 GitHub에서 자동으로 배포(Vercel)가 시작됩니다.
약 1~2분 후 실제 사이트 주소로 접속하면 변경된 내용이 적용된 것을 확인할 수 있습니다.

> **참고**: 로컬(`file://`)에서 테스트할 때도 이 기능을 사용하면 GitHub에 데이터가 저장되므로, 어디서든 최신 데이터를 불러올 수 있습니다.
